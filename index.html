🧠 memharness

Framework-agnostic memory infrastructure for AI agents

Get Started →pip install memharness
📦

10 Memory Types

Conversational, Knowledge Base, Entity, Workflow, Toolbox, Summary, Tool Log, Skills, File, and Persona — each with its own schema, storage strategy, and retrieval pattern.

🔌

Framework Agnostic

Works with LangChain, LangGraph, CrewAI, Deep Agents, or your custom agent framework. One memory layer, any agent. Includes built-in LangChain and LangGraph adapters.

🗄️

Pluggable Backends

PostgreSQL + pgvector for production, SQLite for development, in-memory for testing. Same API across all backends. Swap with one line of config.

⚡

Async-First API

Built for modern Python. Full async/await support throughout. Context managers, type hints, and Pydantic models for configuration.

♻️

Memory Lifecycle

Built-in agents for summarization, consolidation, garbage collection, and entity extraction. Configurable policies keep your memory clean and relevant.

🔍

Self-Exploration Tools

Agents can explore their own memory with built-in LangChain tools. VFS-based tool discovery with tree, ls, grep, and cat operations for the Toolbox memory type.

Quick Start

from memharness import MemoryHarness

async with MemoryHarness("sqlite:///memory.db") as memory:
    # Store a conversation
    await memory.add_conversational("thread1", "user", "Hello!")

    # Add knowledge
    await memory.add_knowledge(
        "Python supports async/await for concurrent programming",
        source="docs"
    )

    # Search semantically
    results = await memory.search_knowledge("concurrency in Python")

    # Assemble context for your agent
    context = await memory.assemble_context("async programming", "thread1")
